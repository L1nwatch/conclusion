# 决策模型设计

Conclusion 的决策工作台用于在形成最终结论前，强迫自己从多个角度检查问题。它保存的是推演过程，不代替最终的一句话结论。

## 设计原则

- 模型是可选工具，不是所有决定都必须完成的审批流。
- 每个模型注册在 SQLite 中，使用稳定 ID、版本和固定问题键，便于网页、搜索和 MCP 读取。
- 保存模型版本，未来调整问题时仍能解释旧记录。
- 原始回答保持简短纯文本；最终核心理由仍可使用 Markdown 附参考链接和公网图片。
- 不把第三方作者的思想包装成逐字、官方或唯一的检查清单。

## 当前内置模型

### `precedent-review`

先检索相似 Conclusion，再回答共同规律、关键差异和本次可复用的原则。这个模型让个人知识库真正形成反馈循环，而不是把“经验”写成脱离历史记录的新感想。

关键词搜索完成前，网页允许手动填写相关 Conclusion 的 ID、标题或链接；FengDock MCP 接入后应先调用 `search_conclusions`。

### `munger-checklist`

问题覆盖风险、独立判断、准备、谦逊、严格分析、资源配置、耐心、果断、适应变化和专注。

这些主题参考《穷查理宝典》的投资原则清单，但提示问题是针对个人决策重新编写的简短版本，不逐字复制、不宣称是官方或唯一版本。

参考：[Poor Charlie's Almanack 官方网站](https://www.poorcharliesalmanack.com/)、[Investment Principles Checklist 主题整理](https://www.sloww.co/investing-principles-checklist-charlie-munger/)

### `scenario-range`

问题：合理范围内的最好、最可能和最坏结果是什么？如果最坏情况发生，如何降低损失或保留退路？

这里的重点不是预测极端事件，而是让上行空间、基准预期和可承受下行同时可见。

### `time-horizons`

问题：这个决定在 10 小时、10 天、10 个月和 10 年后分别意味着什么？

该模型受 Suzy Welch 的 10-10-10 长短期视角启发，但时间单位是 Conclusion 针对个人工作流做的自定义版本，不宣称等同于原方法。

参考：[Suzy Welch 对 10-10-10 的介绍](https://suzywelch.com/books/)

### `inversion`

先问“如果目标是确保失败，我会怎么做”，再找出失败路径、当前暴露和最先需要切断的原因。它与极端思考不同：极端思考描述结果区间，逆向思考寻找造成失败的行为链。

### `inaction-value`

把“不行动”也当作一个真实选项，比较保持现状、避免的成本、错失的价值和未来必须行动的触发点。

### `reversibility`

这是当前唯一额外推荐的必备模型。它先判断决定是难以撤回的单向门，还是容易返回的双向门，再评估退出成本、最小可逆试验和应投入的分析程度。它决定“该继续分析，还是先低成本试一次”。

## 后续扩展

新模型可以通过 HTTP API 或未来 FengDock MCP 创建，不需要修改前端代码。创建时需要：

1. 新增稳定且不可复用的 `modelId`。
2. 提供名称、短标识、说明和 1–20 个带稳定 key 的提示问题。
3. 可选记录来源名称和公网 HTTPS 链接。
4. 由 API 校验长度、问题键唯一性，并以 `version = 1` 保存。
5. 如果修改已有字段含义，创建新版本，不静默覆盖旧数据。

当前暂不继续增加模型。概率校准、决策矩阵、二阶效应和 pre-mortem 都有价值，但与已有模型部分重叠，等真实使用暴露缺口后再添加。MVP 已支持 `list / get / create` 注册表操作，不引入原地更新、删除、自由拖拽节点、复杂分支图或自动 AI 结论。
