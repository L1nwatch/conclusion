# 决策模型设计

Conclusion 的思考模型供 AI/MCP 在形成最终结论前从多个角度检查问题。手动网页保持简洁，不展示模型问卷；模型简析会在详情中转换成 Markdown，不代替最终的一句话结论。

## 设计原则

- 模型定义只有 `name` 和 `explanation` 两个业务字段；稳定 ID 和版本只是内部引用信息。
- 用户要求使用思考模型时，AI 默认把注册表中的全部模型按顺序各过一遍，而不是自行挑选。
- 每个模型只做一小段具体分析，保存在统一的 `answers.analysis`；最终核心理由仍可使用 Markdown 附参考链接和公网图片。
- 旧版多问题回答继续兼容和可读，但新记录不再生成问卷。
- 不把第三方作者的思想包装成逐字、官方或唯一的检查清单。

## 当前内置模型

### `precedent-review`

先检索相似 Conclusion，复用已经验证过的经验，同时指出本次关键差异。FengDock MCP 应先调用 `search_conclusions`；没有先例就明确写“暂无先例”。

### `munger-checklist`

快速检查能力圈、激励、风险、机会成本、事实依据和心理偏差，找出最容易忽略的一项。

这些主题参考《穷查理宝典》的投资原则清单，但提示问题是针对个人决策重新编写的简短版本，不逐字复制、不宣称是官方或唯一版本。

参考：[Poor Charlie's Almanack 官方网站](https://www.poorcharliesalmanack.com/)、[Investment Principles Checklist 主题整理](https://www.sloww.co/investing-principles-checklist-charlie-munger/)

### `scenario-range`

分别看合理的最好、最可能和最坏结果，重点确认最坏结果能否承受，以及能否预留退路。

### `time-horizons`

从 10 小时、10 天、10 个月和 10 年后分别回看这个决定，区分短期情绪和长期影响。

该模型受 Suzy Welch 的 10-10-10 长短期视角启发，但时间单位是 Conclusion 针对个人工作流做的自定义版本，不宣称等同于原方法。

参考：[Suzy Welch 对 10-10-10 的介绍](https://suzywelch.com/books/)

### `inversion`

先问“如果目标是确保失败，我会怎么做”，再检查当前方案是否踩中了这些失败条件。它与极端思考不同：极端思考描述结果区间，逆向思考寻找造成失败的行为链。

### `inaction-value`

把“不行动”也当作一个真实选项，比较它避免的成本、错失的价值和继续等待的代价。

### `reversibility`

判断决定是难以撤回的单向门，还是可以低成本试错的双向门。可逆决定优先做小实验，不可逆决定才投入更多调查。

## 后续扩展

新模型可以通过 HTTP API 或未来 FengDock MCP 创建，不需要修改前端代码。创建时需要：

1. 新增稳定且不可复用的 `modelId`。
2. 只提供名称和简短解释，最多几句话和一个小例子。
3. 由 API 校验长度，并以 `version = 1` 保存。
4. 如果修改已有字段含义，创建新版本，不静默覆盖旧数据。

当前暂不继续增加模型。概率校准、决策矩阵、二阶效应和 pre-mortem 都有价值，但与已有模型部分重叠，等真实使用暴露缺口后再添加。MVP 已支持 `list / get / create` 注册表操作，不引入原地更新、删除、自由拖拽节点、复杂分支图或自动 AI 结论。
